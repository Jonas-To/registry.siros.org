Integration Guide

Use the machine-readable API to automatically configure issuers, verifiers, and wallet applications with credential metadata.

TS11 API

The registry provides a TS11-compliant API at:

https://registry.siros.org/api/v1/schemas.json

This JSON file contains all TS11-compliant credential schemas. See the API Reference for details.

Programmatic Access

The API can be consumed from any language:

JavaScript/Node.js

const response = await fetch(
  'https://registry.siros.org/api/v1/schemas.json'
);
const registry = await response.json();

for (const schema of registry.data) {
  console.log(schema.id, schema.attestationLoS);
}

Python

import requests

registry = requests.get(
    'https://registry.siros.org/api/v1/schemas.json'
).json()

for schema in registry['data']:
    print(schema['id'], schema['attestationLoS'])

Go

resp, _ := http.Get("https://registry.siros.org/api/v1/schemas.json")
defer resp.Body.Close()

var registry struct {
    Data []struct {
        ID             string `json:"id"`
        AttestationLoS string `json:"attestationLoS"`
    } `json:"data"`
}
json.NewDecoder(resp.Body).Decode(&registry)

for _, s := range registry.Data {
    fmt.Println(s.ID, s.AttestationLoS)
}

Use Cases

  • Issuer Configuration - Automatically configure credential issuance with display metadata and claim definitions
  • Verifier Setup - Pre-populate verifier applications with known credential types and validation schemas
  • Wallet Integration - Enable wallets to discover and properly render credentials from multiple issuers
  • CI/CD Pipelines - Fetch latest credential definitions during deployment to keep systems synchronized
