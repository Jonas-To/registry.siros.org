Register a Repository

Add your VCTM definitions to the SIROS Registry by submitting a pull request to include your repository in our list.

Prerequisites

Before registering, ensure your repository meets these requirements:

  1. A vctm branch containing your VCTM files. This branch is automatically created when you set up the mtcvctm GitHub Action.
  2. A registry manifest at .well-known/vctm-registry.json in the vctm branch, listing your VCTM files.
  3. Valid VCTM JSON files conforming to the SD-JWT-VC specification.

Step 1: Set Up Your Repository

The easiest way to prepare your repository is to use the mtcvctm GitHub Action, which automatically:

  • Generates VCTM JSON from Markdown documentation
  • Creates and maintains the vctm branch
  • Builds the .well-known/vctm-registry.json manifest

Alternatively, you can manually create the vctm branch and populate it with your VCTM files.

Step 2: Fork the Registry Repository

  1. Go to github.com/sirosfoundation/registry.siros.org
  2. Click the Fork button to create your own copy
  3. Clone your fork locally: 
    git clone https://github.com/YOUR-USERNAME/registry.siros.org.git
cd registry.siros.org

Step 3: Add Your Repository

Edit the config/repositories.txt file to add your repository: 

# Each line should be: owner/repo
# Lines starting with # are comments

sirosfoundation/mtcvctm
your-org/your-repo        # Add your repository here

The format is owner/repository-name, matching your GitHub URL.

Step 4: Submit a Pull Request

  1. Commit your changes: 
    git add config/repositories.txt
git commit -m "Add your-org/your-repo to registry"
  2. Push to your fork: 
    git push origin main
  3. Go to your fork on GitHub and click New Pull Request
  4. Fill in the PR description with:
    • A brief description of your credential types
    • Link to your repository's vctm branch
    • Any relevant documentation

What Happens Next

After your PR is submitted:

  1. Automated validation: Our CI will verify your repository has a valid vctm branch and registry manifest
  2. Review: A maintainer will review your submission
  3. Merge: Once approved, your PR will be merged
  4. Publication: Within 6 hours, your VCTMs will appear at https://registry.siros.org/your-org/

Registry Manifest Format

Your .well-known/vctm-registry.json should follow this format: 

{
  "name": "Your Organization VCTMs",
  "description": "Verifiable Credential Type Metadata for ...",
  "vctms": [
    {
      "name": "IdentityCredential",
      "path": "credentials/identity.json"
    },
    {
      "name": "MembershipCredential",
      "path": "credentials/membership.json"
    }
  ],
  "commit": "abc123...",
  "timestamp": "2026-02-13T12:00:00Z"
}

Need Help?

If you encounter issues or have questions:

  • Open an issue on the registry repository
  • Check the mtcvctm documentation for the CLI tool
  • Review existing repositories in the registry for examples
