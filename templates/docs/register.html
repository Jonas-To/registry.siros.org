Register a Repository

Add your VCTM definitions to the SIROS Registry by adding the vctm topic to your GitHub repository. The registry automatically discovers and includes repositories with valid VCTM content.

Prerequisites

Before registering, ensure your repository meets these requirements:

  1. A vctm branch containing your VCTM files. This branch is automatically created when you set up the mtcvctm GitHub Action.
  2. A registry manifest at .well-known/vctm-registry.json in the vctm branch, listing your VCTM files.
  3. Valid VCTM JSON files conforming to the SD-JWT-VC specification.

Step 1: Set Up Your Repository

The easiest way to prepare your repository is to use the mtcvctm GitHub Action, which automatically:

  • Generates VCTM JSON from Markdown documentation
  • Creates and maintains the vctm branch
  • Builds the .well-known/vctm-registry.json manifest

Alternatively, you can manually create the vctm branch and populate it with your VCTM files.

Step 2: Add the vctm Topic

The registry automatically discovers repositories by searching for the vctm topic on GitHub. To add this topic:

  1. Go to your repository on GitHub
  2. Click the gear icon (⚙️) next to About in the right sidebar
  3. In the Topics field, type vctm and press Enter
  4. Click Save changes

What Happens Next

After adding the vctm topic:

  1. Discovery: The registry build process searches GitHub for repositories with the vctm topic
  2. Validation: Each discovered repository is checked for a valid vctm branch containing .well-known/vctm-registry.json
  3. Publication: Within 6 hours, your VCTMs will appear at https://registry.siros.org/your-org/

Note: Only repositories with a valid vctm branch and registry manifest will be included. If your repository doesn't appear, verify that your VCTM setup is correct.

Alternative: Submit a Pull Request

If you prefer not to use the topic-based discovery, or need to include a repository without modifying its topics, you can submit a pull request:

  1. Fork github.com/sirosfoundation/registry.siros.org
  2. Edit config/repositories.txt to add your repository: 
    # Format: owner/repo
your-org/your-repo
  3. Submit a pull request with a brief description of your credential types

Registry Manifest Format

Your .well-known/vctm-registry.json should follow this format: 

{
  "name": "Your Organization VCTMs",
  "description": "Verifiable Credential Type Metadata for ...",
  "credentials": [
    {
      "name": "IdentityCredential",
      "vctm_file": "identity.vctm",
      "vct": "https://example.com/credentials/identity"
    },
    {
      "name": "MembershipCredential",
      "vctm_file": "membership.vctm",
      "vct": "https://example.com/credentials/membership"
    }
  ],
  "repository": {
    "commit": "abc123...",
    "url": "https://github.com/your-org/your-repo"
  },
  "generated": "2026-02-13T12:00:00Z"
}

Tip: The mtcvctm GitHub Action generates this manifest automatically from your Markdown credential definitions.

Need Help?

If you encounter issues or have questions:

  • Open an issue on the registry repository
  • Check the mtcvctm documentation for the CLI tool
  • Review existing repositories in the registry for examples
