About the Credential Type Registry

The SIROS Credential Type Registry is a public directory of credential metadata, helping verifiers and wallets discover and understand credential types across SD-JWT VC, mDOC, and W3C VC ecosystems.

What is Credential Metadata?

Credential metadata is standardized information that describes verifiable credentials, enabling interoperability across different implementations. It includes:

  • Display information - names, descriptions, logos, and colors
  • Claim definitions - what data the credential contains
  • Rendering templates - how to visually present the credential
  • Validation rules - which claims are mandatory or selectively disclosable

Supported Formats

The registry supports metadata for multiple credential formats:

  • SD-JWT VC (VCTM) - IETF SD-JWT Verifiable Credentials
  • mso_mdoc - ISO 18013-5 mobile documents (mDL, mID, etc.)
  • W3C VCDM 2.0 - W3C Verifiable Credentials Data Model

Why a Registry?

A public registry enables:

  • Discovery - Verifiers can find metadata for credentials they receive
  • Consistency - Wallets can display credentials correctly using authoritative metadata from issuers
  • Interoperability - A common format and location for credential type information across the ecosystem
  • Documentation - Human-readable descriptions of what each credential type contains and how it should be used

Built by SIROS Foundation

This service is built and maintained by the SIROS Foundation for the digital credentials community.

SIROS Foundation is dedicated to building open infrastructure for secure, privacy-preserving digital identity. We develop open source tools and services that help organizations issue, verify, and manage verifiable credentials.
